## Decisiones tomadas

1. **Migración a pnpm**: Se reemplazó npm por pnpm (v10.33.2 via corepack) por velocidad de instalación, menor uso de disco, y soporte nativo de workspaces. Se creó `.npmrc` con `node-linker=hoisted` (requerido por Expo).
2. **Upgrade a Expo SDK 55**: Se subió de SDK 54 a 55.0.17 (última estable) para usar Expo Router latest, React 19.2.0 y React Native 0.83.6. Validado con `expo-doctor` (18/18 checks).
3. **Entry point**: `"main": "expo-router/entry"` en `package.json` reemplaza `index.ts` → `App.tsx`. Expo Router toma control total del bootstrap.
4. **Scheme configurado**: `"scheme": "vici"` en `app.json` para deep linking (requerido por expo-router).
5. **Estructura de grupos**: `(tabs)` para las 3 pantallas principales con tab bar (Mapa, Actividad, Perfil), `(auth)` para flujo de autenticación con Stack propio. Los grupos no afectan la URL final.
6. **Wallet descartado del MVP**: Tras revisión (grill), se decidió que Wallet no merece tab propio en el MVP. Se implementará como marketplace de merchandising/cupones cuando haya usuarios generando créditos.
7. **Root Layout sin header**: El `Stack` raíz tiene `headerShown: false` — cada grupo maneja sus propios headers. Esto sigue el patrón recomendado por Expo.
8. **Path alias `@/*`**: Configurado en `tsconfig.json` para importar desde `lib/` sin rutas relativas largas. Ejemplo: `@/types/user` en lugar de `../../lib/types/user`.
9. **Rename `src/` → `lib/`**: `src/` sugería ser el código principal pero ya no lo es (`app/` lo es). Se renombró a `lib/` y se eliminaron carpetas vacías de scaffolding (components, hooks, services, store). Solo quedan `types/` y `utils/` con código real.
10. **rtk instalado**: Se configuró rtk-ai/rtk como proxy CLI para comprimir outputs de terminal y ahorrar tokens en asistentes AI.

## Aprendizaje Senior

### File-Based Routing

Expo Router convierte la estructura de archivos en rutas automáticamente, igual que Next.js para web. Cada archivo `.tsx` en `app/` es una ruta. Esto elimina la necesidad de configurar navegación manualmente con `createStackNavigator` o `createBottomTabNavigator`.

**Regla clave**: El directorio `app/` solo contiene rutas y `_layout.tsx`. Nunca co-localices componentes, tipos o utilidades ahí — esos van en `lib/`.

### Grupos de rutas `(parenthesis)`

Los paréntesis crean **grupos lógicos** que no afectan la URL:

- `app/(tabs)/wallet.tsx` → URL final: `/wallet` (no `/tabs/wallet`)
- `app/(auth)/login.tsx` → URL final: `/login` (no `/auth/login`)

Esto permite tener layouts diferentes (tabs vs stack) sin ensuciar las URLs.

### Layouts `_layout.tsx`

Cada `_layout.tsx` define **cómo** se renderizan las rutas hermanas:

- Root `_layout.tsx` → `<Stack>` — decide entre tabs o auth
- `(tabs)/_layout.tsx` → `<Tabs>` — tab bar inferior
- `(auth)/_layout.tsx` → `<Stack>` — navegación lineal con back button

Los layouts se **anidan**: Root Stack > Tabs Layout > Screen. Este pattern de composición es la base de toda la navegación en React Navigation / Expo Router.

### Deep Linking y Scheme

`"scheme": "vici"` en `app.json` habilita URLs como `vici://login` que abren la app directamente en esa pantalla. Expo Router resuelve el routing automáticamente — no se necesita configuración manual de linking.

### Path Aliases

`@/*` en `tsconfig.json` mapea a `./lib/*`. Beneficio: cuando muevas archivos o renombres carpetas, los imports con alias no se rompen (a diferencia de `../../`). Metro (el bundler de React Native) resuelve estos aliases automáticamente en Expo SDK 55.

## Decisiones del grill (revisión de diseño)

- **Wallet fuera del MVP**: Se implementará como marketplace (merchandising, cupones de marcas de running) cuando haya usuarios generando créditos.
- **Mapa como home**: La pantalla principal será un mix: mapa hero card arriba (tocarlo inicia carrera) + feed scrolleable debajo con tarjetas (zonas cercanas, stats, desafíos).
- **Actividad = bitácora**: Historial de carreras, no tracking en vivo. El tracking será un modal full-screen lanzado desde el Mapa.
- **Perfil = identidad de jugador**: Stats, logros, zonas conquistadas, nivel. Settings como sub-pantalla (ícono engranaje).
- **Auth guard**: Redirect condicional en root layout con `useAuth()` (Zustand). Se implementará en la task de auth.
