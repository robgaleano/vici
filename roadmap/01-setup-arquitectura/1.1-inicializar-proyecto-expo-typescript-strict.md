## Objetivo

Levantar la base técnica de VICI con Expo, TypeScript strict, linting, formateo, hooks de Git y una estructura de carpetas que permita crecer sin mezclar todavía navegación, estado global ni backend.

## Decisiones tomadas

1. El bootstrap se hizo con la plantilla `blank-typescript` para no adelantar trabajo de Expo Router, que pertenece a la task 1.2.
2. Se fija `Node 24.11.1` en `.nvmrc` porque el stack de lint actual ya exige `>= 22.13` o `24`, y esa versión está disponible en la máquina.
3. ESLint usa flat config con `eslint-config-expo` y `eslint-config-prettier` para tener compatibilidad directa con Expo 54 y ESLint 9.
4. Husky se limita a `pre-commit` con `lint-staged`; el objetivo aquí es proteger calidad local sin introducir CI todavía.
5. La carpeta `src` nace separada por responsabilidades para que las siguientes tasks aterricen sobre una base estable.

## Aprendizaje Senior

### Generics

En `src/types/api.ts` se han creado tres tipos reutilizables:

- `Nullable<T>` para representar datos opcionales sin repetir `| null`.
- `ApiResponse<T>` para tipar respuestas de servicios sin perder el tipo real de `data`.
- `PaginatedResult<T>` para resultados paginados donde `items` mantiene el tipo concreto de cada elemento.

La idea clave es que el tipo viaja con el dato. Si mañana `T` es `User`, TypeScript puede inferir el shape completo sin casts manuales.

### Utility Types

En `src/types/user.ts` ya hay dos ejemplos prácticos:

- `PublicUser = Pick<User, 'id' | 'username'>` para exponer solo el perfil mínimo.
- `UserPatch = Partial<Pick<User, 'email' | 'username'>>` para updates parciales sin redefinir el modelo.

Esto evita duplicar contratos y reduce drift entre tipos de dominio y tipos de transporte.

### Type Guards

En `src/utils/guards.ts` se ha añadido `isUser(value): value is User`.

Su valor no es académico: cuando Supabase o cualquier servicio externo devuelva `unknown`, el guard te permite validar la forma antes de usarla y conservar type narrowing real dentro del bloque seguro.

## Extras conectados a la task

### Event Loop y Garbage Collection

`strict` no acelera el runtime por sí solo, pero reduce estados inválidos que suelen acabar en ramas defensivas, valores `undefined` inesperados y objetos temporales creados para corregir errores en caliente. En una app con GPS cada pocos segundos, menos ambigüedad de tipos implica menos renders erráticos y menos trabajo accidental en el hilo JS.

### Por qué no meter Expo Router todavía

Separar el bootstrap de la navegación fuerza una disciplina útil: primero dejas resueltos toolchain, convención, estructura y guardrails. Luego añades routing en la task 1.2 sin confundir fallos de arquitectura con fallos de navegación.
