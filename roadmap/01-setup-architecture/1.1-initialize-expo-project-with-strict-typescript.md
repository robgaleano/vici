## Objective

Establish VICI's technical foundation with Expo, strict TypeScript, linting, formatting, Git hooks, and a folder structure that can grow without mixing navigation, global state, or backend concerns too early.

## Decisions Made

1. The bootstrap used the `blank-typescript` template to avoid pulling Expo Router work forward, since that belongs to task 1.2.
2. `Node 24.11.1` is pinned in `.nvmrc` because the current lint stack already requires `>= 22.13` or `24`, and that version is available on the machine.
3. ESLint uses flat config with `eslint-config-expo` and `eslint-config-prettier` for direct compatibility with Expo 54 and ESLint 9.
4. Husky is limited to `pre-commit` with `lint-staged`; the goal here is to protect local quality without introducing CI yet.
5. The `src` folder starts separated by responsibility so the next tasks can land on a stable base.

## Senior Learning

### Generics

Three reusable types were created in `src/types/api.ts`:

- `Nullable<T>` to represent optional data without repeating `| null`
- `ApiResponse<T>` to type service responses without losing the concrete `data` type
- `PaginatedResult<T>` for paginated results where `items` preserves the specific type of each element

The key idea is that the type travels with the data. If tomorrow `T` is `User`, TypeScript can infer the full shape without manual casts.

### Utility Types

There are already two practical examples in `src/types/user.ts`:

- `PublicUser = Pick<User, 'id' | 'username'>` to expose only the minimum profile
- `UserPatch = Partial<Pick<User, 'email' | 'username'>>` for partial updates without redefining the model

That avoids duplicating contracts and reduces drift between domain types and transport types.

### Type Guards

`src/utils/guards.ts` now includes `isUser(value): value is User`.

Its value is not just academic: when Supabase or any external service returns `unknown`, the guard lets you validate the shape before using it and preserve real type narrowing inside the safe block.

## Extras Connected to the Task

### Event Loop and Garbage Collection

`strict` does not speed up runtime by itself, but it reduces invalid states that usually end up creating defensive branches, unexpected `undefined` values, and temporary objects built to patch errors on the fly. In an app receiving GPS updates every few seconds, less type ambiguity means fewer erratic renders and less accidental work on the JS thread.

### Why Expo Router Was Not Added Yet

Separating bootstrap from navigation enforces a useful discipline: first you settle toolchain, conventions, structure, and guardrails. Then you add routing in task 1.2 without confusing architecture issues with navigation issues.
