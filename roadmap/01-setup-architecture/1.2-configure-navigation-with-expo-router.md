## Objective

## Decisions Made

1. **Migration to pnpm**: npm was replaced by pnpm (`v10.33.2` via corepack) for faster installs, lower disk usage, and built-in workspace support. `.npmrc` was created with `node-linker=hoisted` (required by Expo).
2. **Upgrade to Expo SDK 55**: the project moved from SDK 54 to `55.0.17` (latest stable) to use the latest Expo Router, React `19.2.0`, and React Native `0.83.6`. Validated with `expo-doctor` (`18/18` checks).
3. **Entry point**: `"main": "expo-router/entry"` in `package.json` replaces `index.ts` -> `App.tsx`. Expo Router takes full control of bootstrap.
4. **Configured scheme**: `"scheme": "vici"` in `app.json` for deep linking (required by `expo-router`).
5. **Group structure**: `(tabs)` for the 3 main screens with a tab bar (Map, Activity, Profile), `(auth)` for the authentication flow with its own stack. Route groups do not affect the final URL.
6. **Wallet removed from the MVP tabs**: after design review, the decision was that Wallet does not deserve its own tab in the MVP. It will be implemented later as a marketplace for merchandise/coupons once users are generating credits.
7. **Root Layout without header**: the root `Stack` uses `headerShown: false` - each group manages its own headers. This follows Expo's recommended pattern.
8. **`@/*` path alias**: configured in `tsconfig.json` to import from `lib/` without long relative paths. Example: `@/types/user` instead of `../../lib/types/user`.
9. **Rename `src/` -> `lib/`**: `src/` suggested it was still the main code entry, but `app/` now plays that role. It was renamed to `lib/`, and empty scaffold folders (`components`, `hooks`, `services`, `store`) were removed. Only `types/` and `utils/` remain with real code.
10. **`rtk` installed**: `rtk-ai/rtk` was configured as a CLI proxy to compress terminal output and save tokens for AI assistants.
11. **Device validation**: navigation started correctly on iPhone, but Expo Go beta had to be installed because the public App Store build was still pinned to SDK 54.

## Senior Learning

### File-Based Routing

Expo Router turns the file structure into routes automatically, just like Next.js on the web. Every `.tsx` file inside `app/` becomes a route. That removes the need to configure navigation manually with `createStackNavigator` or `createBottomTabNavigator`.

**Key rule**: the `app/` directory should only contain routes and `_layout.tsx`. Never co-locate components, types, or utilities there - those belong in `lib/`.

### Route Groups `(parentheses)`

Parentheses create **logical groups** that do not affect the URL:

- `app/(tabs)/activity.tsx` -> final URL: `/activity` (not `/tabs/activity`)
- `app/(auth)/login.tsx` -> final URL: `/login` (not `/auth/login`)

This lets you keep different layouts (tabs vs stack) without cluttering the URLs.

### `_layout.tsx` Layouts

Each `_layout.tsx` defines **how** its sibling routes are rendered:

- Root `_layout.tsx` -> `<Stack>` - decides between tabs and auth
- `(tabs)/_layout.tsx` -> `<Tabs>` - bottom tab bar
- `(auth)/_layout.tsx` -> `<Stack>` - linear navigation with a back button

Layouts are **nested**: Root Stack > Tabs Layout > Screen. That composition pattern is the foundation of navigation in React Navigation / Expo Router.

### Deep Linking and Scheme

`"scheme": "vici"` in `app.json` enables URLs such as `vici://login`, which open the app directly on that screen. Expo Router resolves routing automatically - no manual linking configuration is required.

### Path Aliases

`@/*` in `tsconfig.json` maps to `./lib/*`. The benefit is that when files move or folders are renamed, alias imports do not break the way `../../` imports do. Metro, the React Native bundler, resolves these aliases automatically in Expo SDK 55.

## Design Review Decisions

- **Wallet out of the MVP tabs**: it will come later as a marketplace (merchandise, running-brand coupons) once users are generating credits.
- **Map as the home screen**: the main screen will be a hybrid layout with a map hero card at the top (tap to start a run) and a scrollable feed below with cards for nearby zones, stats, and challenges.
- **Activity = activity log**: not live tracking. Live tracking will be a full-screen modal launched from the Map.
- **Profile = player identity**: stats, achievements, conquered zones, level. Settings will live as a sub-screen behind a gear icon.
- **Auth guard**: conditional redirect in the root layout with `useAuth()` (Zustand). It will be implemented in the auth task.
